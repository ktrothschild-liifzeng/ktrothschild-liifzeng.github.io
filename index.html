互联网测试开发真的是"点工"吗？

Posted on 2023-11-24 Edited on 2023-12-07

观点1：

观点2：

观点3：

看到脉脉上的一些人对测试的看法，我这边也提供一点我的看法，说的不正确的请斧正！

业务测试手段为啥以点为主？

  • 1、互联网产品迭代太快，业务逻辑简单(toC类型的产品交互必须易上手)，点点点最直接
  • 2、金融类产品业务逻辑更复杂，直接和交易相关，对正确性的要求高，人肉check有保障
  • 3、偏硬件的产品，没有灰度可能，发版后修复成本高，交互路径必须有多轮用例覆盖测试
  • 4、开发为测试构建的基础设施少，特别是内部组件的测试支撑很少考虑，对外只是黑匣子
  • 5、业务测试人员的能力模型普遍是偏沟通推进，以写脚本为主，缺乏系统架构相关的经验
  • 6、测试开发提供的工具易用性差，大部分是玩具级别，稳定性经不住生产环境应用的考验

业务测试真的有手就行？

  • 1、业务测试上手容易做好难，你可能需要有N只手，杂事非常多
    • 流程管理：要跟需求的原型–> 排期 –> 用例编写(可能就是随机发散点点点) –> bug修复验证 –> 测试报告 –> 漏测复盘(甩锅)
  • 2、点很容易，点全很难
    • 有些时候业务的操作组合非常多，很难覆盖全
    • 有些场景的构造需要造数据，需要额外写脚本去调用接口造数据
    • 回归是个大工程，一些主路径不点又怕出问题，点起来又耗时间和精力
    • 用例是个鸡肋，不管你是excel的还是xmind的，费力码了一堆文字，执行时都是点到哪算哪，随机发散先
  • 3、当业务测试leader要八面玲珑
    • 对内要分任务做排期，对外要出数据做统计，费心又费力
    • 出故障要能对外抗压，对内安抚，当然有些就是打嘴炮和甩锅

测试开发做的工具真的那么差？

  • 1、很难一套工具适配所有业务，即使有也会又较高的接入成本，业务不愿意用是常态
  • 2、业务测试有自己的绩效考虑希望在工具层面有产出，至少是想包一层，或名义上共建
  • 3、工具依赖的技术底座不成熟
    • 例如UI自动化
      • Android/iOS手机天然有手机掉线/卡死问题，有UI Hierarchy定位难的问题
      • 带webview的有xpath/ selector的定位因为div的动态变化，animation的影响无法定位问题
      • selenium/ webdriver/ cypress等等基本就是玩具
  • 4、产品化设计考虑不周，造成易用性差
    • 例如接口测试平台
      • 做一个在线版本的postman，接口类似swagger的方式管理，让用户在界面配接口case的各种参数和断言
        • 看上去很理想，但配置界面参数比写脚本来说效率低很多
        • 灵活性差，如果想要上一个接口的返回的某个字段用于后续的接口的请求，即使支持，配置起来也不方便，还不如写脚本

测试开发到底是测试还是开发？

  • 人才过剩，以测开招进来，大部分是还是干点点点的活
  • 纯做工具平台的比例很少，这部分人建议最好做成全栈

卷测试开发到底是卷测试？

  • 人才过剩，没有答案，看机缘巧合
  • 八面玲珑，做业务测试
  • 学历没优势，后端卷不过，不想CRUD，愿意全栈，攻克工具难题，走测开

提供一个工具思路

  • 点一次就同时生成UI的用例和接口的用例，并且能自动运行，是不是就可以提高回归的效率？
    • 以web为例
      • 浏览器有个devtools能够抓取网页的全部流量，就知道待测系统的接口有哪些，以及这些接口请求时的参数
      • 浏览器有个devtools能拿到每个元素的xpath和selector，是不是就可以定位到要点的元素
  • 基于devtools的实现原理做了一个玩具
    • 用例管理
    • UI用例单步调试
    • UI用例录制回放
    • UI用例结果查看
    • 接口用例自动生成回放
    • 接口用例回放结果查看

ffmpeg的常见使用

Posted on 2023-11-23

遇到报错

mp4 Library not loaded: /usr/local/opt/srt/lib/libsrt.1.4.dylib

解决方法

brew install ffmpeg

常见用法

0.5倍速播放转化视频

ffmpeg -i source/video/x-plus.mp4 -an -filter:v “setpts=2.0*PTS” source/video/x-plus-fast-2.mp4

4倍速播放转化视频

ffmpeg -i source/video/x-plus.mp4 -r 16 -an -filter:v “setpts=0.25*PTS” source/video/x-plus-fast-4.mp4

ffmpeg -i input.mov -r 16 -filter:v “setpts=0.125*PTS” -an output.mov

-r指定帧率 -an 静音

4倍声速

ffmpeg -i input.mkv -filter:”atempo=2.0,atempo=2.0” -vn output.mkv

atempo filter 配置区间在0.5和2.0之间，如果需要更高倍，可以使用多个 atempo filter 串在一起来实现

上面的例子是通过2个atempo来实现4倍声速

同时对音频和视频进行速度控制

ffmpeg -i input.mkv -filter_complex “[0:v]setpts=0.5*PTS[v];[0:a]atempo=2.0[a]” -map “[v]” -map “[a]” output.mkv

视频倍速音频2倍速

视频转gif

ffmpeg -i source/video/x-plus-fast-4.mp4 source/gif/x-plus-fast-4.gif

指定区段转

ffmpeg -t 3 -ss 00:00:02 -i small.mp4 small-clip.gif

指定比特率转 -b

ffmpeg -i small.mp4 -b 2048k small.gif

图片转视频

首先要将图片按数字序号命名 1.jpeg -> n.jpeg

ffmpeg -r 10 -f image2 -i %d.jpeg output1.mp4

-r 参数是帧率控制，决定质量

ffmpeg -r 10 -f image2 -i %d.jpeg -b:v 4M output2.mp4

-b:v 决定码率 4M

ffmpeg -f image2 -i %d.jpeg -c:v libx265 output4.mp4

-cv 决定编码格式 libx265

视频格式转换

ffmpeg -i source/video/x-plus-fast-4.mov -vcodec libx264 -preset fast -crf 20 -y -vf “scale=1920:-1” -acodec libmp3lame -ab 128k source/video/x-plus-fast-4.mp4

‘-i filename (input)’ 输入文件名

‘-y (global)’ 覆盖输出文件而不询问

‘-vf filtergraph (output)’ 创建 filtergraph 指定的过滤图，并使用它来过滤流。

‘-vcodec codec (output)’ 设置视频 codec。

‘-acodec codec (input/output)’ 设置音频codec。
