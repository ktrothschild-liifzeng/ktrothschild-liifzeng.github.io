质量工具体系建设随笔

Posted on 2023-11-27 Edited on 2023-11-29

需要服务哪些对象

  • 按端分类

    • 客户端
      • android
      • ios
      • electron
      • qt
    • 服务端
      • openApi
      • webApi
      • 中间件(存储/ cdn/ redis)
      • 全链路
    • web端
      • PC
      • H5
      • 小程序

传统做法:

  • 业务黑盒测试
  • 主要矛盾之一是怎么快速同步需求交互业务逻辑, 快速识别测试点并设计case
  • 主要矛盾之二是怎么才能点的快，回归快，覆盖全
  • UI自动化测试
  • 主要矛盾之一是机器在界面频繁变化的场景下怎么稳定的点
  • 主要矛盾之二是如何增强界面元素的寻找能力
  • 稳定性测试
    • 主要矛盾之一是各种机型如何持久高效地点
    • 主要矛盾之二是crash如何识别去重和聚合
  • 性能测试
    • 主要矛盾之一是如何构造基线benchmark(受到机型/ 网络/ 内容影响)
    • 主要矛盾之二是性能bad case如何辅助分析(systrace，火焰图不够)
  • 接口测试
    • 主要矛盾之一是接口文档可能滞后,有哪些接口需要测试自己抓，可能不全
    • 主要矛盾之二是接口参数太多，编写耗时长
  • 接口压测
    • 主要矛盾之一是接口场景构造
    • 主要矛盾之二是数据脱敏,环境隔离
  • 全链路压测
    • 主要矛盾之一是流量回放时数据脱敏
    • 主要矛盾之二是影子表配套的设计
  • 混沌测试
  • 主要矛盾：混沌的控制和系统配套的监控

部分实践简介:

  • 通过爬取需求/bug元素数据，设计状态机服务来加速流转需求/缺陷
  • 实现点一遍就同时生成UI的Action和net flow的接口测试自动化用例的工具链
  • 分布式 web自动遍历, 在容器中灌浏览器，自动遍历web页面的控件，劫持接口报错，通过控件特征识别前端报错
  • 从swagger生成接口列表，得到接口总数，在页面遍历时自动录制接口，识别动态参数，自动生成接口case
  • 利用Android ContentProvider漏洞重打包注入测试桩获取应用界面布局
  • 分布式 web自动遍历, 在容器中灌浏览器，自动遍历web页面的控件，识别接口报错
  • 混沌chaos系统架构设计
  • 名字服务自动探测(SLB 验证/ 心跳验证/ 故障熔断/ 限流配置/ 恢复机制)

  • 按服务分层(以电商系统为例)

    • 应用层
      • 各种客户端
    • 接入层
      • Api接口测试(http接口测试)
      • 登录/ 注册/ 验证码/ 邮件 服务
    • 业务层
      • 用户服务
        • 注册登录
        • 属性管理
      • 商品服务
        • 库存查询
        • 商品查询
      • 商家服务
        • 货架管理
        • 活动管理
      • 店铺服务
        • 推广服务
      • 搜索服务
        • 商品搜索
        • 店铺搜索
        • 商家搜索
      • 购物车
      • 订单服务
      • 支付服务
      • 物流服务
      • 评论服务
    • 数据流
      • 商品库存同步
      • 商品货架同步
      • 店铺同步
      • 用户事件
      • 订单事件
      • 支付事件
      • 结算事件
      • 评论事件
      • 公共事件
    • 存储层
      • ES
      • ETCD
      • Mysql
      • MongoDB
      • Redis

        如何测试:

  • 电商是典型的分布式微服务架构一定需要分层测试
  • 只有接口测试是不够的，内部的中间件/ 存储都需要做组件测试
  • 如何保障有效灾备(设计一套体系验证服务限流/降级/熔断/异地灾备的有效性)

工具和平台的关系

  • 工具是组件服务，平台是入口和聚合展示的载体
  • 平台还是流程自动流转的载体
  • 工具一定要面向error编程，考虑各阶段可能的异常
  • 工具可以接入到平台的标准是工具是否有丰富的埋点上报来支撑运营优化

ffmpeg的常见使用

Posted on 2023-11-23

遇到报错

mp4 Library not loaded: /usr/local/opt/srt/lib/libsrt.1.4.dylib

解决方法

brew install ffmpeg

常见用法

0.5倍速播放转化视频

ffmpeg -i source/video/x-plus.mp4 -an -filter:v “setpts=2.0*PTS” source/video/x-plus-fast-2.mp4

4倍速播放转化视频

ffmpeg -i source/video/x-plus.mp4 -r 16 -an -filter:v “setpts=0.25*PTS” source/video/x-plus-fast-4.mp4

ffmpeg -i input.mov -r 16 -filter:v “setpts=0.125*PTS” -an output.mov

-r指定帧率 -an 静音

4倍声速

ffmpeg -i input.mkv -filter:”atempo=2.0,atempo=2.0” -vn output.mkv

atempo filter 配置区间在0.5和2.0之间，如果需要更高倍，可以使用多个 atempo filter 串在一起来实现

上面的例子是通过2个atempo来实现4倍声速

同时对音频和视频进行速度控制

ffmpeg -i input.mkv -filter_complex “[0:v]setpts=0.5*PTS[v];[0:a]atempo=2.0[a]” -map “[v]” -map “[a]” output.mkv

视频倍速音频2倍速

视频转gif

ffmpeg -i source/video/x-plus-fast-4.mp4 source/gif/x-plus-fast-4.gif

指定区段转

ffmpeg -t 3 -ss 00:00:02 -i small.mp4 small-clip.gif

指定比特率转 -b

ffmpeg -i small.mp4 -b 2048k small.gif

图片转视频

首先要将图片按数字序号命名 1.jpeg -> n.jpeg

ffmpeg -r 10 -f image2 -i %d.jpeg output1.mp4

-r 参数是帧率控制，决定质量

ffmpeg -r 10 -f image2 -i %d.jpeg -b:v 4M output2.mp4

-b:v 决定码率 4M

ffmpeg -f image2 -i %d.jpeg -c:v libx265 output4.mp4

-cv 决定编码格式 libx265

视频格式转换

ffmpeg -i source/video/x-plus-fast-4.mov -vcodec libx264 -preset fast -crf 20 -y -vf “scale=1920:-1” -acodec libmp3lame -ab 128k source/video/x-plus-fast-4.mp4

‘-i filename (input)’ 输入文件名

‘-y (global)’ 覆盖输出文件而不询问

‘-vf filtergraph (output)’ 创建 filtergraph 指定的过滤图，并使用它来过滤流。

‘-vcodec codec (output)’ 设置视频 codec。

‘-acodec codec (input/output)’ 设置音频codec。
